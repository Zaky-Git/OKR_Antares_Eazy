# Business Rules

## Quarter Rules

- Quarter is auto-generated by system (Q1-Q4 for current year)
- User cannot create, edit, or delete quarters manually
- `POST /periods/ensure-current-year` creates all 4 quarters if not exist
- Dashboard defaults to current quarter (based on current date)
- is_current flag set based on current date matching quarter range

## Ownership Rules

- Only creator (created_by) can edit or delete an objective
- Only creator (created_by) can edit or delete a key result
- Only creator (created_by) OR assignee (assignee_id) can edit or delete an initiative
- All other users get 403 Forbidden

## Progress Calculation

Hierarchy: Objective → Key Result → Initiative (tree)

Rules:
- Leaf Initiative (no children): progress is manual (user updates directly)
- Parent Initiative (has children): progress = average of direct children progress
- Key Result: progress = (current_value / target_value) × 100. Also auto-updated from average root initiative progress when initiatives change.
- Objective: progress = average of key result progress

Key Result metric-based progress:
- KR has `target_value` (goal), `current_value` (actual), and `metric_unit` (optional label like "%", "bugs", "users")
- Progress auto-calculated: (current_value / target_value) × 100, capped at 100
- current_value can also be auto-updated from initiative progress: current_value = (average root initiative progress / 100) × target_value
- User can manually update current_value OR let it auto-calculate from initiatives

Confidence Level:
- Scale 0-10 (0 = will not achieve, 10 = certain to achieve)
- Manually updated by user (subjective assessment)
- Default: 5
- Applied to both Objective and Key Result
- Used for dashboard health overview

Recalculation trigger:
- When a leaf initiative progress is updated → recalculate its parent → recalculate up to root → recalculate Key Result → recalculate Objective
- All in one DB transaction

Manual update restriction:
- If initiative has children, reject manual progress update with 422 status
- Return message: "Cannot manually update progress for parent initiative"

## Status Values

Objective and Key Result:
- PLANNING — default saat baru dibuat
- ON_TRACK — progress berjalan sesuai timeline
- AT_RISK — progress lambat atau ada potensi masalah
- OFF_TRACK — progress sangat tertinggal
- DONE — progress = 100% atau manually set done
- ARCHIVED — objective/KR sudah tidak relevan

Initiative status (manual):
- TODO — belum dikerjakan
- IN_PROGRESS — sedang dikerjakan
- BLOCKED — ada halangan, tidak bisa lanjut
- DONE — selesai
- CANCELLED — dibatalkan, tidak perlu dikerjakan

Status rules:
- Parent initiative status INDEPENDEN dari child (Opsi A)
- Parent blocked = child tetap bisa dikerjakan (blocked hanya label informasi)
- Saat semua child DONE → parent status tetap manual (user harus set DONE sendiri)
- Saat initiative DONE atau CANCELLED → progress slider disabled (tidak bisa diubah lagi)
- Saat initiative BLOCKED → progress slider tetap bisa diubah (karena mungkin partially done)
- User bisa revert status kapan saja (DONE → IN_PROGRESS jika perlu reopen)

## Cascade Delete (Soft Delete)

- Delete Objective → soft-delete all its Key Results → soft-delete all their Initiatives
- Delete Key Result → soft-delete all its Initiatives
- Delete Initiative → soft-delete all child initiatives recursively
- All cascade deletes in one transaction

## Sprint Rules

- Sprint dibuat manual oleh user (semua user bisa create)
- Sprint belongs to a quarter (period_id)
- Sprint start_date dan end_date HARUS dalam range quarter yang dipilih (validasi backend)
- Hanya boleh 1 sprint ACTIVE per quarter pada satu waktu
- Initiative bisa dibuat tanpa sprint (sprint_id nullable)
- Initiative tanpa sprint tetap bisa di-plan tapi belum masuk execution cycle
- Initiative bisa di-assign ke sprint kapan saja (PATCH update sprint_id)
- Child initiative bisa punya sprint berbeda dari parent
- Saat sprint di-close (status → COMPLETED), user mengisi review_note dan retro_note
- Initiative yang belum selesai saat sprint di-close tetap di sprint itu, bisa di-reassign ke sprint berikutnya
- Semua user bisa close sprint (no role restriction di MVP)

Sprint status flow:
- PLANNING → ACTIVE → COMPLETED

Sprint closure:
- review_note = catatan hasil kerja ("apa yang selesai")
- retro_note = catatan perbaikan ("apa yang perlu diperbaiki di sprint berikutnya")
- Kedua field opsional saat close, tapi UI encourage isi

Validasi:
- start_date harus >= period.start_date
- end_date harus <= period.end_date
- start_date harus < end_date
- Tidak boleh activate sprint jika sudah ada sprint ACTIVE di quarter yang sama

## Notification Rules

Create web notification when initiative meets ALL conditions:
1. Status is NOT DONE and NOT CANCELLED
2. Has assignee_id (not null)
3. Has due_date (not null)
4. Due date matches a reminder threshold

Reminder keys and timing:
- DUE_H3 — due date is 3 days from now
- DUE_H1 — due date is 1 day from now (tomorrow)
- DUE_TODAY — due date is today
- OVERDUE — due date has passed

## Duplicate Notification Prevention

- Before creating notification, check notification_logs table
- If record exists with same (initiative_id, user_id, reminder_key) → skip
- If not exists → create notification AND insert notification_log
- This ensures each reminder is sent only once per initiative per user

## Dashboard Summary (per period)

- Total Objectives: count of objectives in period
- Average Progress: average of all objectives progress
- Average Confidence: average confidence_level across objectives
- On Track: objectives with status ON_TRACK
- At Risk: objectives with status AT_RISK
- Off Track: objectives with status OFF_TRACK
- Overdue Initiatives: initiatives with due_date < today AND status NOT DONE/CANCELLED
- Recent Updates: latest initiative_updates (limit 10)

## Annual Dashboard (per year)

Annual Summary:
- Total Objectives (all quarters combined)
- Overall Average Progress (all quarters)
- Overall Average Confidence
- Total Completed Objectives (status DONE)
- Completion Rate = (completed / total) × 100%
- Total Initiatives
- Total Overdue Initiatives

Quarter Overview (Q1-Q4 side-by-side):
- Per quarter: total objectives, avg progress, avg confidence, status breakdown (on_track, at_risk, off_track, done)
- Allows comparing performance across quarters in one view

API: GET /api/dashboard/annual?year=2026
Returns both annual_summary and quarters array with per-quarter stats
